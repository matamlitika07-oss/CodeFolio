You are an expert full-stack developer. Build a complete web application called **CodeFolio** — a no-code portfolio builder for developers. Think "Linktree meets Squarespace, but built for engineers."

## TECH STACK
- Frontend: React.js (Vite), React Router v6, React Hook Form, React Helmet
- Backend: Node.js + Express.js
- Database: MongoDB + Mongoose
- Email: Nodemailer (mock with console.log if no SMTP)
- Styling: Plain CSS with CSS variables (no Tailwind, no Bootstrap)
- Auth: JWT-based (email + password), token in localStorage

## PHASE 2: BACKEND API

### server/index.js
- Express app, connect to MongoDB via MONGODB_URI env var
- CORS enabled for localhost:5173
- Mount routes: /api/auth, /api/portfolio, /api/public
- Listen on PORT env var or 5000

### server/middleware/authMiddleware.js
- Verify JWT from Authorization: Bearer <token> header
- Attach decoded user id to req.userId
- Return 401 if missing or invalid

### server/routes/auth.js
POST /api/auth/register
  - Validate username, email, password present
  - Check username and email not already taken
  - Hash password with bcrypt (saltRounds: 10)
  - Create and save User
  - Return JWT signed with JWT_SECRET (expires in 7d) and { username, email }

POST /api/auth/login
  - Find user by email
  - Compare password with bcrypt
  - Return JWT and { username, email }

### server/routes/portfolio.js (all routes protected by authMiddleware)
GET /api/portfolio/me
  - Return full user object (exclude password)

PUT /api/portfolio/profile
  - Body: profile fields + contactEmail + templateId
  - Update and return saved user

PUT /api/portfolio/projects
  - Body: { projects: [...] }
  - Replace projects array and save

PUT /api/portfolio/skills
  - Body: { skills: { frontend, backend, devops, other } }
  - Replace skills and save

PUT /api/portfolio/template
  - Body: { templateId }
  - Update templateId and save

POST /api/portfolio/contact/:username
  - Body: { senderName, senderEmail, message }
  - Find user by username
  - Use Nodemailer (transporter with EMAIL_USER, EMAIL_PASS env vars) to send email to user.contactEmail
  - If no SMTP configured, console.log the message instead
  - Never return contactEmail in response
  - Return { success: true }

### server/routes/public.js
GET /api/public/:username
  - Find user by username
  - If not found return 404
  - Return user data excluding password and contactEmail
